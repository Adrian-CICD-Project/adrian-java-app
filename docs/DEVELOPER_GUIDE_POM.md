# pom.xml Configuration for New Applications

This document describes what developers need to configure in `pom.xml` to properly deploy a new application in the CI/CD pipeline.

## Required Elements

### 1. Basic Project Metadata

```xml
<groupId>com.example</groupId>           <!-- Change to your groupId -->
<artifactId>app-name</artifactId>         <!-- Application name -->
<version>0.0.1-SNAPSHOT</version>
<name>app-name</name>
<description>Application description</description>
```

### 2. Java Version

```xml
<properties>
    <java.version>21</java.version>  <!-- Must match Dockerfile -->
</properties>
```

### 3. Required Dependencies

```xml
<!-- Testing -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>

<!-- Prometheus Metrics (optional but recommended) -->
<dependency>
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>
```

### 4. Required Plugins

```xml
<build>
    <plugins>
        <!-- Spring Boot plugin - JAR packaging -->
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>

        <!-- JaCoCo - code coverage (required by SonarQube) -->
        <plugin>
            <groupId>org.jacoco</groupId>
            <artifactId>jacoco-maven-plugin</artifactId>
            <version>0.8.11</version>
            <executions>
                <execution>
                    <goals><goal>prepare-agent</goal></goals>
                </execution>
                <execution>
                    <id>report</id>
                    <phase>verify</phase>
                    <goals><goal>report</goal></goals>
                </execution>
            </executions>
        </plugin>

        <!-- SonarQube plugin -->
        <plugin>
            <groupId>org.sonarsource.scanner.maven</groupId>
            <artifactId>sonar-maven-plugin</artifactId>
            <version>4.0.0.4121</version>
        </plugin>
    </plugins>
</build>
```

## Pre-deployment Checklist

- [ ] Changed `groupId` and `artifactId` to appropriate values
- [ ] Set correct Java version (must match Dockerfile)
- [ ] Added `spring-boot-starter-test` for testing
- [ ] Added `jacoco-maven-plugin` for coverage
- [ ] Added `sonar-maven-plugin` for code analysis
- [ ] Removed duplicate dependencies and plugins

## CI/CD Notes

1. **SBOM** - automatically generated by `cyclonedx-maven-plugin` (invoked from command line)
2. **Trivy scan** - scans Docker image, no pom.xml changes required
3. **SonarQube** - requires plugin and proper JaCoCo configuration
